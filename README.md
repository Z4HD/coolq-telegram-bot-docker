每次构建都会从[jqqqqqqqqqq/coolq-telegram-bot](https://github.com/jqqqqqqqqqq/coolq-telegram-bot)的master分支拉取最新的代码。

# Docker.com 一般的准则和建议
- [ ] 容器应该是短暂的
- [ ] 使用.dockerignore文件
- [ ] 避免安装不必要的包
- [ ] 每个容器只运行一个进程
- [ ] 最小化层的数量
- [ ] 排序多行参数
